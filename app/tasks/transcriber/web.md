### Текущие проблемы

- Веб-бекенд шлет в ML любой текст (логи и пр.) вместо четких JSON команд

- Нет handshake (инициализации) с ML сервисом

- Используется локальная транскрибация, хотя есть ML сервис

- Нет корректной обработки ошибок подключения к ML

### Требования к WebSocket `/ws/stream` (связь с фронтендом)

**1. Инициализация сессии**

- При подключении фронтенда ничего не отправлять, пока фронтенд сам не инициализирует сессию

- Дождаться от фронтенда JSON: `{"type": "init", "config": {"language": null}}`

- Подключиться к ML сервису по адресу `ML_WS_URL` (эндпоинт `/transcriber/ws/transcribe`)

- Отправить ML сервису этот же init JSON

- Дождаться от ML ответа `init_ack`

- Только после этого пересылать аудио данные

**2. Форвардинг фронтенд → ML**

|Тип сообщения|Действие|
|-|-|
|Бинарные данные (bytes)|Отправить в ML как bytes|
|JSON команды (`end`, `ping`, `cancel`)|Отправить в ML как текст|
|Любой другой текст (не JSON)|Игнорировать и залогировать ошибку|

**3. Форвардинг ML → фронтенд**

- Все JSON сообщения от ML пересылать фронтенду как есть (без изменений)

- Бинарные сообщения от ML игнорировать

**4. Обработка ошибок подключения к ML**

|Ситуация|Действие|
|-|-|
|ML недоступен при подключении|Отправить фронтенду `{"type": "error", "code": "ML_UNAVAILABLE"}` и закрыть WS|
|Разрыв соединения с ML во время работы|Отправить фронтенду `{"type": "error", "code": "ML_DISCONNECTED"}`и закрыть WS|
|Таймаут ожидания данных от ML|Отправить фронтенду `{"type": "error", "code": "TIMEOUT"}` и закрыть WS|

**5. Удалить устаревший код**

- Удалить функции: `_run_local_transcribe`, `_get_whisper_model`, `_extract_audio_to_wav`

- Удалить импорт `faster_whisper`

- Удалить переменную `_WHISPER_MODEL`

- Удалить все упоминания `language="ru"`

- Упростить `/api/upload` (убрать локальную транскрибацию) или удалить эндпоинт, если весь функционал переведен на стриминг

**6. Healthcheck**

- Обновить `/health`: добавить поле `ml_available: true/false`

- Обновить `/check-ml`: использовать новый протокол (отправить init на `/transcriber/ws/transcribe`, дождаться `init_ack`)

### Форматы сообщений от ML (пересылаются фронтенду как есть)

|Тип|Пример|
|-|-|
|`init_ack`|`{"type":"init_ack","status":"ready","session_id":"123"}`|
|`language_info`|`{"type":"language_info","detected_language":"en","confidence":0.98}`|
|`transcript`|`{"type":"transcript","start_ms":0,"end_ms":2500,"text":"Hello","is_final":true}`|
|`pong`|`{"type":"pong"}`|
|`cancel_ack`|`{"type":"cancel_ack","status":"cancelled"}`|
|`error`|`{"type":"error","code":"WHISPER_ERROR","message":"..."}`|

### Критерии приемки

- Веб-бекенд не шлет в ML никакой текст кроме JSON команд

- Реализован handshake: init → init_ack перед началом передачи аудио

- При недоступности ML фронтенд получает понятную ошибку

- Удален весь код локальной транскрибации (WhisperModel)

- Все JSON от ML пересылаются фронтенду без изменений

### Комментарий

`/ws/stream` — вход для фронтенда. Внутри подключается к ML на `/transcriber/ws/transcribe`. Убедись что `ML_WS_URL` указывает на правильный адрес с пространством `/transcriber/`. Не шли в ML текстовые логи - только JSON команды. Добавь try/catch при подключении к ML.
